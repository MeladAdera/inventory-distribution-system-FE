## 📋 Overview

### Purpose
The Clients admin page (`/clients`) is the primary interface for warehouse managers to view, search, filter, and manage client accounts. It provides full CRUD operations (add, edit, delete, view) through modal dialogs without leaving the page.

### Business Value
- Single-screen client management: create, edit, delete
- Client avatar with auto-generated initials — no image uploads required
- Active/inactive status badge for quick visual health check
- Client-side search across name + phone with instant feedback
- Credentials info banner clarifies that login details are auto-generated by the system
- Bilingual (AR/EN) with full RTL/LTR layout support
- 650 ms skeleton load simulation ready to swap for a real API hook

## 🧩 Component Reference

### `ClientAvatar`
```tsx
<ClientAvatar name="بقالة الواحة" size={34} />
// Renders: 34×34px circle, bg ink-900, text amber-500
// Initials: first letter of each word, max 2 chars
// Font size scales at 40% of size prop (~13px at default size)
```

### `ClientStatusBadge`
```tsx
<ClientStatusBadge status="active" label="نشط" />
// Renders: dot + label pill
```

Status → colour mapping:

| Status | Dot / text | Background |
|--------|-----------|------------|
| `active` | `success-700` | `success-100` |
| `inactive` | `ink-400` | `ink-200/60` |

### `ClientsTableCard`

Receives paginated clients + filter state + callbacks. Owns the toolbar UI, skeleton, and pagination UI.

Grid columns: `40px 2fr 1.2fr 1fr 1fr 1fr 1fr 120px`

| Column | Content | Notes |
|--------|---------|-------|
| # | Row index | Mono, `ink-400`, hidden mobile |
| Client | Avatar + name | Truncated, `ink-900` |
| Phone | Phone number | Mono `xs`, `ink-600`, forced `ltr` |
| Address | City name | `ink-600` |
| Products | Count | Mono, `ink-700` |
| Last activity | Relative time string | `ink-500` |
| Status | `ClientStatusBadge` | — |
| Actions | Eye / Pencil / Trash2 | Right-aligned |

### `ClientFormModal`

- `mode: 'add' | 'edit'` controls title and pre-filling.
- Fields: Arabic name (required), English name, Phone (required, ltr), Address, Notes (textarea).
- Credentials banner: amber-50 background, key-round icon, auto-generation message.
- Required fields: Arabic name, Phone.

### `ClientDeleteConfirmModal`

Backdrop click is **intentionally disabled** — user must choose Delete or Cancel explicitly. Mirrors the products `DeleteConfirmModal` pattern.

## 🔧 State & Data Flow

```
page.tsx
  │
  ├── clients: AdminClient[]       ← useState(MOCK_CLIENTS), updated by CRUD handlers
  ├── isLoading: boolean           ← setTimeout(650ms) on mount, simulates fetch
  ├── search / statusFilter        ← useState, reset page on change
  ├── page: number                 ← pagination cursor
  └── modal: ModalState            ← discriminated union: none | add | edit | view | delete
       │
       ├── filtered = useMemo(...)   ← searches name (AR+EN) and phone (strips spaces)
       └── paginated = filtered.slice(...)
```

### CRUD Operations (local state, swappable with API mutations)

| Action | Handler | Effect |
|--------|---------|--------|
| Add | `handleAdd(data)` | Appends `AdminClient` with next id, status `active`, `product_count: 0` |
| Edit | `handleEdit(client, data)` | Updates name, phone, address, notes by id |
| Delete | `handleDelete(client)` | Filters out by id |

### Search Logic

```typescript
const matchSearch =
  !search ||
  name.toLowerCase().includes(search.toLowerCase()) ||
  c.phone.replace(/\s/g, '').includes(search.replace(/\s/g, ''));
```

Phone search strips whitespace from both sides — "050412" matches "050 412 8830".
